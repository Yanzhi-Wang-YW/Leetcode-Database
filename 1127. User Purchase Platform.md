```md
1127. User Purchase Platform  
Table: Spending

+-------------+---------+
| Column Name | Type    |
+-------------+---------+
| user_id     | int     |
| spend_date  | date    |
| platform    | enum    | 
| amount      | int     |
+-------------+---------+
The table logs the history of the spending of users that make purchases from an online shopping website that has a desktop and a mobile application.
(user_id, spend_date, platform) is the primary key (combination of columns with unique values) of this table.
The platform column is an ENUM (category) type of ('desktop', 'mobile').
 

Write a solution to find the total number of users and the total amount spent using the mobile only, the desktop only, and both mobile and desktop together for each date.

Return the result table in any order.

The result format is in the following example.

 

Example 1:

Input: 
Spending table:
+---------+------------+----------+--------+
| user_id | spend_date | platform | amount |
+---------+------------+----------+--------+
| 1       | 2019-07-01 | mobile   | 100    |
| 1       | 2019-07-01 | desktop  | 100    |
| 2       | 2019-07-01 | mobile   | 100    |
| 2       | 2019-07-02 | mobile   | 100    |
| 3       | 2019-07-01 | desktop  | 100    |
| 3       | 2019-07-02 | desktop  | 100    |
+---------+------------+----------+--------+
Output: 
+------------+----------+--------------+-------------+
| spend_date | platform | total_amount | total_users |
+------------+----------+--------------+-------------+
| 2019-07-01 | desktop  | 100          | 1           |
| 2019-07-01 | mobile   | 100          | 1           |
| 2019-07-01 | both     | 200          | 1           |
| 2019-07-02 | desktop  | 100          | 1           |
| 2019-07-02 | mobile   | 100          | 1           |
| 2019-07-02 | both     | 0            | 0           |
+------------+----------+--------------+-------------+ 
Explanation: 
On 2019-07-01, user 1 purchased using both desktop and mobile, user 2 purchased using mobile only and user 3 purchased using desktop only.
On 2019-07-02, user 2 purchased using mobile only, user 3 purchased using desktop only and no one purchased using both platforms
```

```py
import pandas as pd 

def user_purchase(spending: pd.DataFrame) -> pd.DataFrame:
    user_activity = spending.groupby(['spend_date', 'user_id']).agg(
        platforms=('platform', lambda x: set(x)), 
        amount=('amount', 'sum')
    ).reset_index() 
    def create_platform_column(platform):
        if len(platform) == 2: 
            return 'both'
        if 'mobile' in platform: 
            return 'mobile' 
        if 'desktop' in platform: 
            return 'desktop' 
    user_activity['platform'] = user_activity['platforms'].apply(create_platform_column) 
    user_activity = user_activity.groupby(['spend_date', 'platform']).agg(
        total_amount=('amount', 'sum'),
        total_users=('user_id', 'count') 
    ).reset_index() 
    # # continuous calendar date range
    # min_date = pd.to_datetime(user_activity['spend_date'].min()).strftime('%Y-%m-%d') 
    # max_date = pd.to_datetime(user_activity['spend_date'].max()).strftime('%Y-%m-%d') 
    # dates = pd.date_range(start=min_date, end=max_date)
    # unique dates 
    dates = spending['spend_date'].unique() 
    platforms = ['both', 'desktop', 'mobile']
    index = pd.MultiIndex.from_product([dates, platforms], names=['spend_date', 'platform'])
    master_table = pd.DataFrame(index=index).reset_index() 
    output = master_table.merge(user_activity, how='left', on=['spend_date', 'platform']) 
    output['total_amount'] = output['total_amount'].fillna(0)  
    output['total_users'] = output['total_users'].fillna(0)  
    return output
``` 
