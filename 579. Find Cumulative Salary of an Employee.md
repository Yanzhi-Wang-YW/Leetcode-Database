```md
579. Find Cumulative Salary of an Employee  
Table: Employee

+-------------+------+
| Column Name | Type |
+-------------+------+
| id          | int  |
| month       | int  |
| salary      | int  |
+-------------+------+
(id, month) is the primary key (combination of columns with unique values) for this table.
Each row in the table indicates the salary of an employee in one month during the year 2020.
 

Write a solution to calculate the cumulative salary summary for every employee in a single unified table.

The cumulative salary summary for an employee can be calculated as follows:

For each month that the employee worked, sum up the salaries in that month and the previous two months. This is their 3-month sum for that month. If an employee did not work for the company in previous months, their effective salary for those months is 0.
Do not include the 3-month sum for the most recent month that the employee worked for in the summary.
Do not include the 3-month sum for any month the employee did not work.
Return the result table ordered by id in ascending order. In case of a tie, order it by month in descending order.

The result format is in the following example.

 

Example 1:

Input: 
Employee table:
+----+-------+--------+
| id | month | salary |
+----+-------+--------+
| 1  | 1     | 20     |
| 2  | 1     | 20     |
| 1  | 2     | 30     |
| 2  | 2     | 30     |
| 3  | 2     | 40     |
| 1  | 3     | 40     |
| 3  | 3     | 60     |
| 1  | 4     | 60     |
| 3  | 4     | 70     |
| 1  | 7     | 90     |
| 1  | 8     | 90     |
+----+-------+--------+
Output: 
+----+-------+--------+
| id | month | Salary |
+----+-------+--------+
| 1  | 7     | 90     |
| 1  | 4     | 130    |
| 1  | 3     | 90     |
| 1  | 2     | 50     |
| 1  | 1     | 20     |
| 2  | 1     | 20     |
| 3  | 3     | 100    |
| 3  | 2     | 40     |
+----+-------+--------+
Explanation: 
Employee '1' has five salary records excluding their most recent month '8':
- 90 for month '7'.
- 60 for month '4'.
- 40 for month '3'.
- 30 for month '2'.
- 20 for month '1'.
So the cumulative salary summary for this employee is:
+----+-------+--------+
| id | month | salary |
+----+-------+--------+
| 1  | 7     | 90     |  (90 + 0 + 0)
| 1  | 4     | 130    |  (60 + 40 + 30)
| 1  | 3     | 90     |  (40 + 30 + 20)
| 1  | 2     | 50     |  (30 + 20 + 0)
| 1  | 1     | 20     |  (20 + 0 + 0)
+----+-------+--------+
Note that the 3-month sum for month '7' is 90 because they did not work during month '6' or month '5'.

Employee '2' only has one salary record (month '1') excluding their most recent month '2'.
+----+-------+--------+
| id | month | salary |
+----+-------+--------+
| 2  | 1     | 20     |  (20 + 0 + 0)
+----+-------+--------+

Employee '3' has two salary records excluding their most recent month '4':
- 60 for month '3'.
- 40 for month '2'.
So the cumulative salary summary for this employee is:
+----+-------+--------+
| id | month | salary |
+----+-------+--------+
| 3  | 3     | 100    |  (60 + 40 + 0)
| 3  | 2     | 40     |  (40 + 0 + 0)
+----+-------+--------+
```

```py
import pandas as pd

def cumulative_salary(employee: pd.DataFrame) -> pd.DataFrame: 
    # remove most recent month 
    employee = employee.sort_values(['id', 'month'], ascending=[True, False]) 
    employee['rn'] = employee.groupby('id').cumcount() + 1 
    employee = employee.loc[employee['rn'] != 1]   
    employee = employee.drop(columns=['rn']) 
    if employee.empty: 
        return employee 
    # fill month 
    def month_mapping(group):
        if group['month'].nunique() == 1:
            return group[['id', 'month']] 
        min_month = group['month'].min() 
        max_month = group['month'].max() 
        return pd.DataFrame({
            'id': group['id'].iloc[0],
            'month': list(range(min_month, max_month + 1))  
        }) 
    fill_month = employee.groupby('id', group_keys=False).apply(month_mapping).reset_index(drop=True)     
    # output 
    output = fill_month.merge(employee, how='left', on=['id', 'month']) 
    output['salary'] = output['salary'].fillna(value=0) 
    output = output.sort_values(['id', 'month'], ascending=[True, True]) 
    output['rolling_sum'] = output.groupby('id')['salary'].transform(lambda x: x.rolling(window=3, min_periods=1).sum()) 
    output = employee.merge(output, how='inner', on=['id', 'month']) 
    output = output[['id', 'month', 'rolling_sum']] 
    output = output.rename(columns={'rolling_sum': 'Salary'})                    
    return output 
``` 

## Microsoft Fabric — Dataflow Gen2 (Step-by-Step)

> The goal is to compute a **rolling 3-month salary sum** per employee, excluding each employee's most recent month, using the Power Query (M) editor inside a Dataflow Gen2 in Microsoft Fabric.

---

### Step 1 — Create a New Dataflow Gen2
1. Open your **Microsoft Fabric workspace**.
2. Click **+ New** → **Dataflow Gen2**.
3. In the Power Query editor, click **Get data** and connect to your `Employee` table (Lakehouse, Warehouse, SQL, CSV, etc.).

---

### Step 2 — Sort Rows
1. Select the `id` column → **Sort Ascending**.
2. Then select the `month` column → **Sort Descending**.

This ensures each employee's records are ordered newest-first.

---

### Step 3 — Remove Each Employee's Most Recent Month
1. **Add Column → Index Column → From 0**, grouped by resetting per employee:
   - Go to **Add Column → Custom Column** and use this M expression:
     ```m
     // Assigns a row number per employee (0-based, newest month = 0)
     ```
   - Alternatively, use **Group By**:
     - Group by `id`.
     - New column name: `AllRows`, Operation: **All Rows**.
     - Then expand using a custom column:
       ```m
       = Table.AddIndexColumn([AllRows], "rn", 0, 1)
       ```
     - Expand `AllRows` to get `month`, `salary`, and `rn`.
2. **Filter** the `rn` column: remove rows where `rn = 0` (this drops the most recent month per employee).
3. **Remove** the `rn` column — it is no longer needed.

**Shortcut (M formula bar):**
```m
let
    Sorted = Table.Sort(Employee, {{"id", Order.Ascending}, {"month", Order.Ascending}}),
    Grouped = Table.Group(Sorted, {"id"}, {{"AllRows", each _, type table}}),
    AddIndex = Table.TransformColumns(Grouped, {{"AllRows", each Table.AddIndexColumn(Table.Sort(_, {{"month", Order.Descending}}), "rn", 0, 1)}}),
    Expanded = Table.ExpandTableColumn(AddIndex, "AllRows", {"month", "salary", "rn"}),
    Filtered = Table.SelectRows(Expanded, each [rn] <> 0),
    RemovedRn = Table.RemoveColumns(Filtered, {"rn"})
in
    RemovedRn
```

---

### Step 4 — Fill Missing Months (Required for Correct Rolling Window)

Months with no record must be present as `salary = 0` so the rolling window of 3 works correctly.

1. **Group By** `id`, keeping **All Rows**.
2. Add a **Custom Column** that generates the full month range per employee and left-joins the salary data:
   ```m
   = let
       minM = List.Min([AllRows][month]),
       maxM = List.Max([AllRows][month]),
       allMonths = Table.FromList({minM..maxM}, Splitter.SplitByNothing(), {"month"}, null, ExtraValues.Error),
       joined = Table.NestedJoin(allMonths, {"month"}, [AllRows], {"month"}, "Data", JoinKind.LeftOuter),
       expanded = Table.ExpandTableColumn(joined, "Data", {"salary"}),
       filled = Table.ReplaceValue(expanded, null, 0, Replacer.ReplaceValue, {"salary"})
   in
       filled
   ```
3. Expand the result and sort by `id` ascending, `month` ascending.

---

### Step 5 — Compute the Rolling 3-Month Sum

Power Query does not have a built-in rolling/window function, so use one of these approaches:

#### Option A — Self-Merge (No Custom M)
1. Create **two references** of the filled table.
2. **Merge** the table with itself on `id`, using a **Custom Column** merge condition where the right-side `month` is in `{[month], [month]-1, [month]-2}`.
3. After expanding, **Group By** `id` and `month`, summing `salary` from the merged rows.

#### Option B — Custom Column with `List.Sum` (Recommended)
Add a **Custom Column** with this M expression:
```m
= List.Sum(
    Table.SelectRows(
        FilledTable,
        (r) => r[id] = [id] and r[month] >= [month] - 2 and r[month] <= [month]
    )[salary]
)
```
Name this column `Salary` (the rolling 3-month sum).

#### Option C — Advanced Editor (Full M Script)
```m
let
    Source = Employee,
    Sorted = Table.Sort(Source, {{"id", Order.Ascending}, {"month", Order.Descending}}),
    Grouped = Table.Group(Sorted, {"id"}, {{"AllRows", each _, type table}}),
    AddRn = Table.TransformColumns(Grouped, {{"AllRows", each Table.AddIndexColumn(_, "rn", 0, 1)}}),
    Expanded = Table.ExpandTableColumn(AddRn, "AllRows", {"month", "salary", "rn"}),
    RemovedRecent = Table.SelectRows(Expanded, each [rn] <> 0),
    DroppedRn = Table.RemoveColumns(RemovedRecent, {"rn"}),

    // Fill missing months
    Grouped2 = Table.Group(DroppedRn, {"id"}, {{"AllRows", each _, type table}}),
    Filled = Table.TransformColumns(Grouped2, {{"AllRows", each
        let
            minM = List.Min(_[month]), maxM = List.Max(_[month]),
            full = Table.FromList({minM..maxM}, Splitter.SplitByNothing(), {"month"}),
            joined = Table.NestedJoin(full, {"month"}, _, {"month"}, "D", JoinKind.LeftOuter),
            exp = Table.ExpandTableColumn(joined, "D", {"salary"}),
            rep = Table.ReplaceValue(exp, null, 0, Replacer.ReplaceValue, {"salary"})
        in rep
    }}),
    ExpandFilled = Table.ExpandTableColumn(Filled, "AllRows", {"month", "salary"}),
    SortedFinal = Table.Sort(ExpandFilled, {{"id", Order.Ascending}, {"month", Order.Ascending}}),

    // Rolling 3-month sum
    AddRolling = Table.AddColumn(SortedFinal, "Salary", each
        List.Sum(
            Table.SelectRows(SortedFinal, (r) => r[id] = [id] and r[month] >= [month] - 2 and r[month] <= [month])[salary]
        ), Int64.Type),

    // Keep only months the employee actually worked (inner join back)
    Original = Table.SelectRows(Table.RemoveColumns(RemovedRecent, {"rn"}), each true),
    Final = Table.NestedJoin(Original, {"id","month"}, AddRolling, {"id","month"}, "R", JoinKind.Inner),
    Result = Table.ExpandTableColumn(Final, "R", {"Salary"}),
    Output = Table.Sort(Result, {{"id", Order.Ascending}, {"month", Order.Descending}}),
    Cleaned = Table.SelectColumns(Output, {"id", "month", "Salary"})
in
    Cleaned
```

---

### Step 6 — Remove Helper Columns & Set Destination
1. Keep only `id`, `month`, and `Salary`.
2. Sort by `id` ascending, then `month` descending.
3. Click **+ Add data destination** → choose your **Lakehouse table**, **Warehouse**, or **KQL Database**.
4. Map columns and click **Publish**.
