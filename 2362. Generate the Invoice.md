```md
Table: Products

+-------------+------+
| Column Name | Type |
+-------------+------+
| product_id  | int  |
| price       | int  |
+-------------+------+
product_id is the primary key for this table.
Each row in this table shows the ID of a product and the price of one unit.
 

Table: Purchases

+-------------+------+
| Column Name | Type |
+-------------+------+
| invoice_id  | int  |
| product_id  | int  |
| quantity    | int  |
+-------------+------+
(invoice_id, product_id) is the primary key for this table.
Each row in this table shows the quantity ordered from one product in an invoice. 
 

Write an SQL query to show the details of the invoice with the highest price. If two or more invoices have the same price, return the details of the one with the smallest invoice_id.

Return the result table in any order.

The query result format is shown in the following example.

 

Example 1:

Input: 
Products table:
+------------+-------+
| product_id | price |
+------------+-------+
| 1          | 100   |
| 2          | 200   |
+------------+-------+
Purchases table:
+------------+------------+----------+
| invoice_id | product_id | quantity |
+------------+------------+----------+
| 1          | 1          | 2        |
| 3          | 2          | 1        |
| 2          | 2          | 3        |
| 2          | 1          | 4        |
| 4          | 1          | 10       |
+------------+------------+----------+
Output: 
+------------+----------+-------+
| product_id | quantity | price |
+------------+----------+-------+
| 2          | 3        | 600   |
| 1          | 4        | 400   |
+------------+----------+-------+
Explanation: 
Invoice 1: price = (2 * 100) = $200
Invoice 2: price = (4 * 100) + (3 * 200) = $1000
Invoice 3: price = (1 * 200) = $200
Invoice 4: price = (10 * 100) = $1000

The highest price is $1000, and the invoices with the highest prices are 2 and 4. We return the details of the one with the smallest ID, which is invoice 2. 
```

### 1. Base Data: `Purchases`

This is the original dataset showing the items and quantities belonging to various invoice IDs.

| invoice_id | product_id | quantity |
| --- | --- | --- |
| 1 | 1 | 2 |
| 3 | 2 | 1 |
| 2 | 2 | 3 |
| 2 | 1 | 4 |
| 4 | 1 | 10 |

### 2. Base Data: `Products`

This is the lookup table containing the unit prices for each product.

| product_id | price |
| --- | --- |
| 1 | 100 |
| 2 | 200 |

### 3. Step: `Purchase Deatail` (Merged & Calculated)

This query joins the `Products` table to the `Purchases` table so the unit `price` is available. It then adds a custom column (`total_price_individual_invoice`) that multiplies the `quantity` by the unit `price` to get the line-item total.

| invoice_id | product_id | quantity | price | total_price_individual_invoice |
| --- | --- | --- | --- | --- |
| 1 | 1 | 2 | 100 | 200 |
| 3 | 2 | 1 | 200 | 200 |
| 2 | 2 | 3 | 200 | 600 |
| 2 | 1 | 4 | 100 | 400 |
| 4 | 1 | 10 | 100 | 1000 |

### 4. Step: `Highest Price` (Grouped, Sorted & Filtered)

Branching off the previous step, this query groups the data by `invoice_id` and sums the line-item totals to find the overall total for each invoice.
*(Note: Invoice 2 and Invoice 4 tie with a total of $1,000. Because the script sorts by total price descending and then by `invoice_id` ascending, Invoice 2 wins the tiebreaker).* The top row is kept, and all columns except `invoice_id` are removed.

| invoice_id |
| --- |
| 2 |

### 5. Final Output: `Invoice Detail`

The final query merges the winning `invoice_id` (from Step 4) back with the `Purchase Deatail` table (from Step 3). This isolates just the line items for Invoice 2. Finally, the calculated total column is renamed to `price` and the `invoice_id` is removed to produce a clean receipt.

| product_id | quantity | price |
| --- | --- | --- |
| 2 | 3 | 600 |
| 1 | 4 | 400 |
