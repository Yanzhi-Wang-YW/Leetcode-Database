3657. Find Loyal Customers
Table: customer_transactions

+------------------+---------+
| Column Name      | Type    | 
+------------------+---------+
| transaction_id   | int     |
| customer_id      | int     |
| transaction_date | date    |
| amount           | decimal |
| transaction_type | varchar |
+------------------+---------+
transaction_id is the unique identifier for this table.
transaction_type can be either 'purchase' or 'refund'.
Write a solution to find loyal customers. A customer is considered loyal if they meet ALL the following criteria:

Made at least 3 purchase transactions.
Have been active for at least 30 days.
Their refund rate is less than 20% .
Refund rate is the proportion of transactions that are refunds, calculated as the number of refund transactions divided by the total number of transactions (purchases plus refunds).

Return the result table ordered by customer_id in ascending order.

The result format is in the following example.

 

Example:

Input:

customer_transactions table:

| 4              | 101         | 2024-02-20       | 250.00 | purchase         |
| 5              | 102         | 2024-01-10       | 100.00 | purchase         |
| 6              | 102         | 2024-01-12       | 120.00 | purchase         |
| 7              | 102         | 2024-01-15       | 80.00  | refund           |
| 8              | 102         | 2024-01-18       | 90.00  | refund           |

# Write your MySQL query statement below
WITH purchase AS (
    SELECT customer_id, transaction_type, 
            (CASE 
                WHEN transaction_type = "purchase" THEN 1 
                ELSE 0 
            END) As check_purchase 
    FROM customer_transactions     
), 
count_purchase AS (
    SELECT customer_id, SUM(check_purchase) AS total_purchase    
    FROM purchase 
    GROUP BY customer_id 
    HAVING total_purchase >= 3      
), 
count_days AS (
    SELECT customer_id, TIMESTAMPDIFF(DAY, MIN(transaction_date), MAX(transaction_date)) AS active_days   
    FROM customer_transactions 
    GROUP BY customer_id 
    HAVING active_days >= 30    
),
join_table_one AS (
    SELECT count_purchase.customer_id 
    FROM count_purchase 
    INNER JOIN count_days 
    ON count_purchase.customer_id = count_days.customer_id    
), 
check_refund AS (
    SELECT customer_id, transaction_type, 
            (CASE 
                WHEN transaction_type = "refund" THEN 1 
                ELSE 0 
            END) As check_refund
    FROM customer_transactions     
), 
total_to_refund AS (
    SELECT customer_id, COUNT(*) AS total_transaction, SUM(check_refund) AS refund_transaction  
    FROM check_refund 
    GROUP BY customer_id      
), 
refund_rate AS (
    SELECT customer_id, (refund_transaction / total_transaction) AS refund_ratio 
    FROM total_to_refund
), 
filter_refund_rate AS (
    SELECT * 
    FROM refund_rate 
    WHERE refund_ratio < 0.2    
), 
join_table_two AS (
    SELECT join_table_one.customer_id
    FROM join_table_one 
    INNER JOIN filter_refund_rate 
    ON join_table_one.customer_id = filter_refund_rate.customer_id     
) 
SELECT * 
FROM join_table_two 
ORDER BY customer_id ASC
